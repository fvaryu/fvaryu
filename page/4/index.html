关于Android消息机制你所需要知道的

发表于 2016-08-04 | 分类于 Android

Android的消息机制几乎是面试必问的话题，当然也并不是因为面试，而去学习，更重要的是它在Android的开发中是必不可少的，占着举足轻重的地位，所以弄懂它是很有必要的。下面就来说说最基本的东西。

Looper

作用：

  • 关联起Thread
  • 循环取出消息

1、Looper是否可以直接实例化？

Looper构造方法是私有的，其中做了两件事

  • 创建一个MessageQueue
  • 得到与之对应的Thread

2、一个线程能对应多个Lopper？

不能，一个线程对应一个Looper对象，通过ThreadLocal保证一个线程只有一个Looper与之对应，如果多次调用Looper.prepare();则会抛出运行时异常。

3、Looper是无限循环，会阻塞吗？

是，当开启一个loop后是一个死循环，从MessageQueue中取出消息，处理消息，但是也有可能退出，在没有消息后退出循环。

4、可以再次调用Looper.prepareMainLooper吗？

不可以，Looper.prepareMainLooper最终也是调用prepare()，同2.

5、MainLooper什么时候创建的？

MainLooper是启动Activity创建ActivityThread(并不是一个Thread)时候创建，所以不能多次创建。

Handler

作用:

  • 发送消息到MessageQueue
  • 处理消息

1、Handler如何与Looper、MessageQueue关联起来？

我们知道一个Looper对应一个Thread，一个Looper包含一个MessageQueue。当我们创建Handler时就会从当前线程中取出与之对应的Looper，让后在从Looper中取出MessageQueue。

Message

单项链表结构。

作用：

  • 数据的载体

1、消息如何复用的？

从全局消息池(链表结构)中

2、Message为什么能传递？

Android中想要传递对象要么实现Serializable要么Parcelable，在这里是实现了Parcelable接口。

3、如何与Handler关联？

我们知道在消息传机制中Handler充当着“快递员”的角色，那么他又是如何与“货物”－－Message发生关系呢？实际上Message有一个成员变量target他的类型正是Handler，

当我们通过Handler去send一个Message时候最终都会为target赋值为this，即当前的Handler。

另为如果是通过Message.Obtain(),获取的复用Message也会为其赋值。

多说一句，Handler.obtainMessage()调用的就是Message.Obtain()。

总结：

通过一系列的包涵关系，最终Looper、Handler、Message、MessageQueue即发生关联，从而形成一个闭合，开启消息循环。

Gradle，深入浅出（一）

发表于 2016-08-02 | 分类于 Android ， Gradle

Gradle已经出来很久了，相信你一定有所了解，作为Android开发者，如果不知道，或者不了解那就真的Out了。那么Gradle什么呢？又什么用处呢？

简述

什么是Gradle？

Gradle 是新一代的自动化构建工具，它是一个独立的工具，跟 Android和Android Studio 没有关系，换句话说，AS是依赖于Gradle，官方网站, 它同 Ant、Maven这类构建工具的不同在于它是一种DSL(区域描述语言)，什么是DSL？说白了就是黑话，比如警察卧底，做手势，标记通知对接人，对接人就明白是什么意思。而Ant、Maven基于 xml 来进行描述的，不够简洁，同是也不能包含一些复杂的逻辑在里面，而 Gradle 采用的是一种叫做 Groovy 的语言，兼容Java语言，实际上是Java的一种脚本化的体现，都是要运行在JVM中的，但它是一种动态语言，在 Java 基础上做了不少改进，用起来更加简洁、灵活，而且 Gradle 完全兼容 Maven、Ivy，这点基本上宣布了 Maven、Ivy 可以被抛弃了，Gradle 的推出主要以 Java 应用为主，当然目前还支持 Android、C、C++。相信在以后会又更大的运用空间。

有什么用？

我们都知道Android开发语言是Java，而Java是一种编译语言，就是要编译器编译成运行环境能够识别的东西，在Java中就是class字节码，Android中略有不同是打包在一起的class－－dex。所以如何能更好的把源文件组织起来，就成了问题。比如Android从源代码、资源文件到最后的Apk，是需要Android SDK资源包里面的工具来完成（见下图）。

Gradle和Android的关系

Gradle实际上和Android或者AS没有任何关系，Gradle是可以单独安装在电脑上使用的(安装过程自行学习）。完成可打开命令行输入 Gradle -v查看是否安装正确。

其中可以看到：Gradle的版本:V2.8（现在已经2.14了），Groovy的版本:V2.4.4，虚拟机JVM：V1.8.0(Java8)，是不是感觉很熟悉呢，就是安装JDK的步骤了。

既然没有关系那为什么能够，帮助我们管理好项目呢？（这个问题问得好），实际上Gradle基于Groovy但它并不直接参与到具体项目的工作流上来，而仅仅是定义了API，就和我们平日调用Android API一样，真正起作用的是：基于Groovy开发出来的插件。比如Java插件－－专门编译Java工程的，Groovy插件等等。
